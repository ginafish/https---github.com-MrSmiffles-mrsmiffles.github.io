Version 2.0 vs. Version 1.0

When making calls with the BLS API, you must select between using Version 1.0 calls and Version 2.0 calls.

When to use Version 1.0:

  • You are only planning to send 25 or fewer queries per day.
  • You are only going to include 25 or less serieses per query.
  • You will only ask for 10 or less years per query.
  • You are not looking for aggregate statistics like net/percent changes, optional annual averages, and series description information from the catalog.

With Version 2.0:

  • You are required to make an account and receive an API key to use Version 2.0. After registering, check your e-mail for a confirmation e-mail that will include your API key. Click here to register.
  • You get up to 500 queries per day.
  • You can include up to 50 serieses per query, and can call data for up to 20 years' worth per query.
  • You can query for net/percent changes, optional annual averages, and series description information from the catalog.
  • Version 2.0 also supports x-www-form-urlencoded content type for the request body, as well as excel spreadsheet (XLSX), but the latter is outside the scope of this guide.

Basically, you get more features with Version 2.0 than Version 1.0, but you may not want to go to the trouble of registering for an API key if you're only doing a few simple queries.
